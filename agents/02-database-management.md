Master relational and NoSQL databases, from schema design to query optimization, building systems that handle billions of operations.

## What You'll Master

### 1. Relational Databases (SQL)
- **PostgreSQL**: Advanced features, JSON support, window functions
- **MySQL/MariaDB**: Scalability, replication, performance tuning
- **Oracle/SQL Server**: Enterprise features, advanced optimization
- **SQLite**: Embedded databases, mobile apps

### 2. SQL Fundamentals & Advanced
- **Basic CRUD**: SELECT, INSERT, UPDATE, DELETE
- **Joins**: INNER, LEFT, RIGHT, FULL, CROSS, self-joins
- **Aggregation**: GROUP BY, HAVING, aggregate functions
- **Advanced**: Window functions, CTEs, subqueries, recursive queries
- **Optimization**: Query plans, indexes, statistics

### 3. Data Modeling & Design
- **Conceptual**: Entity-relationship diagrams (ERD)
- **Logical**: Relationships, normalization rules
- **Physical**: Table design, column selection, constraints
- **Normalization**: 1NF through BCNF, denormalization trade-offs
- **Dimensional**: Star schema, snowflake schema for analytics

### 4. Indexing Strategy
- **B-tree indexes**: Most common, efficient for range queries
- **Hash indexes**: Exact match queries
- **Bitmap indexes**: Low cardinality columns
- **Full-text indexes**: Text search
- **Covering indexes**: Reduce disk I/O
- **Composite indexes**: Multiple columns
- **Index optimization**: Cost-benefit analysis

### 5. Query Optimization
- **EXPLAIN/ANALYZE**: Understanding execution plans
- **Index selection**: Right index for right query
- **Query rewriting**: Transform queries for efficiency
- **N+1 problem**: Batch operations
- **Materialized views**: Precomputed results
- **Partitioning**: Large table management

### 6. NoSQL Databases
- **Document stores**: MongoDB, Firestore
- **Key-value stores**: Redis, Memcached, DynamoDB
- **Time-series**: InfluxDB, Prometheus, TimescaleDB
- **Graph databases**: Neo4j, ArangoDB
- **Search engines**: Elasticsearch, Solr

### 7. Transactions & Concurrency
- **ACID properties**: Atomicity, Consistency, Isolation, Durability
- **Isolation levels**: Read uncommitted, committed, repeatable read, serializable
- **Locks**: Row locks, table locks, deadlock detection
- **Optimistic locking**: Version numbers
- **Pessimistic locking**: Database locks

### 8. Scaling & High Availability
- **Replication**: Master-slave, master-master setups
- **Sharding**: Horizontal partitioning, shard keys
- **Federation**: Multiple independent databases
- **Caching**: Query result caching, Redis integration
- **Read replicas**: Scale read operations
- **Backup & recovery**: Point-in-time recovery
