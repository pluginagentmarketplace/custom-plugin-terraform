## What You'll Master

### 1. Design Patterns (23 Gang of Four)
- **Creational**: Singleton, Factory, Abstract Factory, Builder, Prototype
- **Structural**: Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy
- **Behavioral**: Observer, Strategy, Command, State, Template Method, Chain of Responsibility, Iterator, Mediator, Memento, Visitor, Interpreter

### 2. Architectural Styles
- **Monolithic**: Single codebase, simple but less flexible
- **Microservices**: Independent services, complex but scalable
- **Serverless**: Function-based, minimal ops, cost-effective
- **Event-driven**: Asynchronous, loosely coupled
- **Layered**: Horizontal layers, traditional approach

### 3. System Design Principles
- **Scalability**: Handle growth in users/data
- **Availability**: System uptime and reliability
- **Consistency**: Data correctness
- **Performance**: Response time and throughput
- **Maintainability**: Code organization
- **Extensibility**: Add features easily

### 4. Core Concepts
- **Load balancing**: Distribute requests
- **Caching**: In-memory, CDN, application-level
- **Database scaling**: Replication, sharding
- **Asynchronism**: Queues, message brokers
- **Service communication**: REST, gRPC, messaging

### 5. Large-scale System Design
- **Twitter-scale**: Billions of tweets, massive throughput
- **Facebook-scale**: Social graphs, real-time updates
- **Netflix-scale**: Streaming, personalization
- **Uber-scale**: Real-time location, matching
- **YouTube-scale**: Video storage, transcoding
