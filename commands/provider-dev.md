# /provider-dev

Build custom Terraform providers for any API or cloud platform.

## Provider Development Path

### Phase 1: Setup (1 week)
- Set up development environment
- Go 1.18+ installed
- Terraform Plugin SDK v2
- Create scaffolding
- Hello world provider

### Phase 2: Basic Provider (2 weeks)
- Provider configuration
- Authentication setup
- First resource (CRUD)
- Basic testing
- Documentation

### Phase 3: Multiple Resources (3-4 weeks)
- Add more resources
- Data sources
- Resource dependencies
- Error handling
- Comprehensive testing

### Phase 4: Polish & Release (2 weeks)
- 100% test coverage
- Documentation
- Registry preparation
- Versioning
- Release process

## Resource Implementation

```go
Create: Create resource
Read:   Read resource from API
Update: Update resource
Delete: Delete resource
```

## Testing Strategy

- Unit tests for logic
- Acceptance tests for API
- Test all CRUD operations
- Test error scenarios
- Import testing
- Parallel testing

## Publishing

- GitHub repository
- Terraform Registry
- Documentation
- Examples
- Semantic versioning
- Release automation

## Popular Custom Providers

- Custom APIs
- Internal cloud platforms
- SaaS platforms
- Third-party services

- Proprietary tools

## Getting Started

1. Review Terraform SDK docs
2. Study existing providers
3. Plan your resources
4. Start coding
5. Test thoroughly
6. Publish to registry

Get help with `/terraform-mentor provider`
